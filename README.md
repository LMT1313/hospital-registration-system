# 🏥 医院挂号诊疗管理系统 (C/S架构)

## 📖 项目简介
本系统是一款基于 **C/S (Client/Server) 架构** 开发的医院挂号诊疗综合管理平台。系统采用 **Qt** 作为客户端开发框架，结合 **MySQL** 关系型数据库，针对患者、医生、护士、管理员四类核心角色的业务需求进行了精细化的功能模块划分。

各业务模块紧密衔接，实现了从挂号预约、医生诊疗、护士协助到后台管理的全流程闭环管理。所有涉及信息修改的操作均通过事务机制实时同步至 MySQL 数据库，严格保障医疗数据的一致性、准确性与安全性。

---

## 🛠️ 技术栈
* **客户端框架**: Qt (C++)
* **数据库**: MySQL
* **架构模式**: C/S (Client/Server)

---

## 🧩 系统功能模块
系统共划分为五大核心模块，覆盖医院日常诊疗全场景：

### 1. 🔐 登录模块
* **统一入口**：提供系统统一的身份验证入口。
* **角色路由**：用户输入账号、密码和身份类型，验证通过后自动跳转至对应角色（患者/医生/护士/管理员）的专属工作台。
* **安全机制**：身份验证失败提供明确错误提示，保障账号安全。

### 2. 🧑‍🦱 患者模块
围绕挂号预约、信息查询及个人管理展开，是系统的核心用户端：
* **预约挂号**：支持按科室、医生、坐诊时间段筛选，提交挂号申请并生成订单，支持在线缴费。
* **号源查询**：实时展示全院科室号源，包括医生姓名、坐诊时间、剩余号数、挂号费用及挂号时段。
* **科室医生查询**：查阅科室基础信息（简介、位置、诊疗范围）及医生详细信息（职称、专业领域、擅长方向、排班等）。
* **我的订单**：分类查看挂号订单（待就诊/已就诊/已取消），并关联展示缴费记录（金额、时间、支付方式）。
* **个人信息管理**：支持查看与修改个人基本信息（姓名、联系方式、身份证、就诊卡等）及登录密码。

### 3. 👨‍⚕️ 医生模块
围绕号源管理、患者诊疗与医嘱处理，提升门诊工作效率：
* **号源发布**：医生自主发布指定时间段的坐诊号源，设置可约数量、费用及科室，支持对已发布号源的修改、暂停和删除。
* **患者信息查看**：一键查看预约患者的基本信息、既往病历及本次挂号订单详情。
* **诊疗工作**：开具电子处方、检查/检验申请单，记录诊疗意见，所有信息自动关联至患者电子病历。
* **医嘱管理**：录入、编辑并保存医嘱（用药指导、术后护理、复查提醒等），数据实时同步至患者病历。

### 4. 👩‍⚕️ 护士模块
作为医生诊疗的辅助模块，提供分诊、协助及缴费支持：
* **患者信息查看**：查阅本科室就诊患者的基本信息、挂号订单、病历及医嘱信息。
* **协助诊疗**：协助完成诊疗前基础检查（体温、血压、心率等），数据录入系统并实时同步至患者诊疗记录。
* **缴费管理**：协助患者完成线下挂号及诊疗费用缴费，更新系统订单缴费状态，并支持打印缴费凭证。

### 5. ⚙️ 管理员模块
拥有系统最高权限，负责整体运维、用户及数据安全管理：
* **用户管理**：对患者、医生、护士进行增删改查，包括添加新用户、删除无效账号、修改信息及重置密码。
* **权限管理**：为医生、护士分配操作权限（如科室权限、功能操作权限），支持角色权限的动态增删改。
* **数据管理**：定期备份核心数据（患者信息、订单、病历等），支持故障恢复及数据批量导出。
* **系统设置**：配置系统基础参数（挂号基准费用、号源发布规则、缴费方式等），维护科室基础信息（增删改）。

## 💾 数据库设计亮点
* **实时同步**：所有前端 UI 操作产生的数据变更均通过 Qt 的 `QSqlQuery` / `QSqlTableModel` 实时写入 MySQL。
* **事务控制**：关键业务（如挂号扣费、医嘱保存）采用数据库事务，确保数据强一致性。
